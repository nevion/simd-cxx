simd-cxx. simple. elegant. efficient.

simd-cxx v1.0 is an implementation of the latest C++ API for DDS as specified by the OMG DDS-PSM-C++ standard. This standard heavily drew concepts and inspiration from simd-cxx v0.x as well as from other experimental DDS APIs such as escalier.

The result is a simple, elegant and efficient API that I am sure you will enjoy. To wet your appetite, let's see how you might write a simple twitter application with the new API.

Let's start by defining our Twit topic type:

struct Twit_t {
     string userid;
     string msg;
 };
 #pragma keylist Twit userid

Let's write now the fragment of the application to post Twits:

// Create a domain participant to join the proper DDS domain
DomainParticipant dp(twitterID);
// Create a publisher in the given domain
Publisher pub(dp);
// Create a Twit topic in the domain
Topic<Twit_t> twit(dp, "Twit");
// Create a Twit DataWriter
DataWriter<Twit_t> dw(pub, twit);
// Twit...
dw << Twit("acorsaro", "Have fun with #simd-cxx !");

The fragment of the application to read Twits is pretty similar:

// Create a domain participant to join the proper DDS domain
DomainParticipant dp(twitterID);
// Create a publisher in the given domain
Subscriber sub(dp);
// Create a Twit topic in the domain
Topic<Twit_t> twit(dp, "Twit");
// Create a Twit DataWriter
DataReader<Twit_t> dr(sub, twit);
// Read Twits
LoanedSamples<Twit_t> samples = dr.read();
// Iterate over the read samples and print the latest twits.
for_each(samples.begin(), samples.end(), printTwit);
